# User Interviews Log

## Interview 1 — Prakhar Pandey, Executive at Alt Tab Media

**Current AI stack:** ChatGPT Team, Claude Pro, Canva AI tools, Gemini

**Pain points:**
Prakhar said their team keeps trying different AI tools for content writing, thumbnails, and marketing tasks, but after a few months nobody remembers which subscription is actually useful.

**Direct quotes:**

* “We keep buying tools because every week there’s a new AI product.”
* “Sometimes two people use different tools for the same work.”
* “At the end of the month the AI bill looks bigger than expected.”

**Main insight:**
He cared more about understanding where money is going than getting a perfect savings number.

**What it changed:**
I added clearer recommendation reasons and focused more on making the audit easy to understand instead of only showing savings.

## Interview 2 — Himanshu Chauhan, Integrated M.Tech AIML Student at VIT Bhopal

**Current AI stack:** ChatGPT Plus, Gemini, GitHub Copilot Student Pack

**Pain points:**
Himanshu said students and developers often try multiple AI tools together because different tools are better for coding, notes, and assignments.

**Direct quotes:**

* “I use ChatGPT for explanations but Copilot for coding.”
* “Sometimes I forget which subscription is active.”
* “I would use something like this if it showed which tools overlap.”

**Main insight:**
Not all overlap is bad. Some users intentionally keep multiple tools because each one solves a different problem.

**What it changed:**
I made the recommendation engine less aggressive so it suggests “consider switching” instead of forcing users to remove tools.

## Interview 3 — Friend Running Small Freelance Projects

**Current AI stack:** ChatGPT Plus, OpenAI API, Claude Free

**Pain points:**
He mainly struggled with API costs because usage changes every month depending on client work.

**Direct quotes:**

* “API billing feels unpredictable.”
* “Subscriptions are easier to track than token usage.”
* “Sometimes I spend more on APIs than actual subscriptions.”

**Main insight:**
People want simple and understandable pricing recommendations, not complicated token calculations.

**What it changed:**
I improved the API overspend recommendations and added simpler summary explanations in the audit results page.
