# User Interviews Log

## Interview 1 — A.K., CTO, Seed-Stage Dev Tools Startup (9 employees)
**Current AI stack:** Cursor Business (7 seats), GitHub Copilot Business (9 seats), ChatGPT Team (8 seats), Claude Pro (2 seats), OpenAI API for a small internal script. 
**Pain points:** Tool overlap and inconsistent ownership of spend. He could not explain who was paying for what without digging into four different receipts. 
**Direct quotes:**
- "We buy tools ad hoc and suddenly the bill is 1k a month. No one feels accountable."
- "Copilot and Cursor both help the same five engineers. It feels redundant, but no one wants to be the one to cut it."
- "We are paying for Claude Pro for two PMs even though they rarely use it after launch week."
**Surprising insight:** He preferred a single, finance-friendly recommendation list over a generic "save money" score. The contradiction was that he admitted they overspend, but said he still would not downgrade a tool if it slowed the team by 5 percent, even if it saved $300/month. 
**What it changed:** I added a stronger per-tool rationale in recommendations and focused on savings and operational impact. This directly influenced the one-sentence reasons and confidence labels on each recommendation card.

## Interview 2 — N.R., Founder, Bootstrapped Agency (3 employees)
**Current AI stack:** ChatGPT Plus (1 seat), Claude Pro (1 seat), OpenAI API (usage-based for internal workflows), Gemini Advanced (1 seat). 
**Pain points:** Volatile API costs. They set a budget but still get surprise spikes when a client asks for a bulk analysis. 
**Direct quotes:**
- "The API is great but it turns into a black box. I only notice when the bill hits."
- "I keep Plus because I like the UI, but the real work runs through the API."
- "I need to bill clients for usage. Subscriptions feel hard to allocate back." 
**Surprising insight:** They intentionally keep both API and subscription because API spend is billable to clients, while the subscription is for internal use. That was a useful contradiction: they accept overlap when it helps client accounting. 
**What it changed:** I made the API overspend rule recommend a subscription only when it is clearly cheaper for internal usage, and I kept the language neutral ("consider switching") rather than forcing a downgrade.

## Interview 3 — S.L., Head of Data, Series A Analytics SaaS (45 employees)
**Current AI stack:** Claude Team (20 seats), GitHub Copilot Enterprise (30 seats), Gemini Advanced (5 seats), Anthropic API for data labeling, ChatGPT Team (10 seats). 
**Pain points:** Procurement and compliance. They want fewer vendors to simplify security review and avoid duplicated contracts.
**Direct quotes:**
- "Every new vendor is a security review. I would rather consolidate even if the cost is slightly higher."
- "I need something I can show finance without them asking ten follow-up questions."
- "We keep Gemini for one team because they like the Workspace integration. It is hard to take away."
**Surprising insight:** They cared more about vendor consolidation and compliance than raw savings. They also viewed a polished PDF or shareable report as essential for internal buy-in.
**What it changed:** I emphasized the shareable audit URL and ensured the AI summary is formal and finance-friendly. I also planned a week 2 PDF export as a natural extension.
